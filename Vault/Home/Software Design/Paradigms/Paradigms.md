# Intro

A *paradigm* here is a way of modelling and building software — the mental model you adopt before writing code. Knowing several lets you pick the simplest model for the job: an object-oriented style organises behaviour around state and identity, a functional style shines in data transformations where immutability cuts bugs, and an event-driven one decouples producers from consumers. This folder groups the **programming paradigms** that shape how you structure the code itself (OOP, functional, event-driven).
